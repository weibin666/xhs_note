# 小红书运营自动化平台 — 开发计划

> 定位:**自用 / 小团队**,**纯合规**(不做模拟登录、不做自动发布、不做批量养号)。
> 核心壁垒在 **AI 内容引擎**;发布走官方后台,平台只做"内容生产 + 选题 + 管理 + 分析"。

---

## 一、关键约束与取舍

小红书没有开放的官方运营 API,因此:

| 模块 | 合规做法 | 说明 |
|------|----------|------|
| AI 内容生产 | ✅ 直接做 | 调用 DeepSeek,核心价值所在 |
| 选题 / 爆款拆解 | ✅ 直接做 | 用户手动贴爆款链接/正文,AI 解析 |
| 发布管理 | ⚠️ 改造 | **不自动发布**;做草稿 / 排期 / 内容日历 + 到点提醒,人工去官方后台发 |
| 数据分析 | ⚠️ 二期 | 无 API → **手动导入**(创作者中心导出 / 手动录入),不爬取 |

---

## 二、技术栈

- **后端**:Python + FastAPI(异步、易接 LLM API)
- **AI**:DeepSeek(OpenAI 兼容接口,`openai` SDK)
  - `deepseek-chat` — 内容生成 / 爆款拆解 / 检测复检等(支持 JSON 输出模式)
  - 模型与 base_url 走环境变量,可随时切换其他兼容服务
- **前端**:Next.js (App Router) + TypeScript + Tailwind CSS
- **存储**:SQLite 起步(自用够用),后续可换 Postgres
- **封面生成**(后期):HTML/CSS 模板 + Playwright 截图渲染
